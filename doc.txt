====================================================================
1) OVERVIEW
====================================================================

This backend is an Express + MongoDB application for URL shortening with optional user authentication.

Core capabilities:
- Create short URLs (guest users allowed).
- Create short URLs tied to authenticated users.
- Redirect from short URL to original URL.
- User registration/login/logout with JWT in HTTP-only cookies.
- Fetch all URLs created by a logged-in user.

Architecture style:
- Layered architecture:
  - Routes (HTTP mapping)
  - Controllers (request/response orchestration)
  - Services (business logic)
  - DAO (database access)
  - Models (Mongoose schemas)
  - Middleware + Utils (cross-cutting concerns)

12) MIDDLEWARE
====================================================================

File: BACKEND/src/middleware/auth.middleware.js

Purpose:
- Strict authentication guard for protected routes.

Behavior:
- Reads token from req.cookies.accessToken
- If no token -> 401 Unauthorized
- Verifies token -> gets decoded ID
- Finds user by ID
- If missing/invalid -> 401
- Else attaches req.user and next()

Error response shape:
- { message: "Unauthorized" }
- On catch includes error object too.

------------------------------------------------------------

File: BACKEND/src/utils/attachUser.js

Purpose:
- Optional authentication middleware for all requests.

Behavior:
- Reads accessToken cookie
- If absent -> next()
- If present, verify token and fetch user
- If valid user, sets req.user
- Never blocks request if invalid; just continues

Why useful:
- Allows endpoint behavior to adapt based on login state without forcing auth.
